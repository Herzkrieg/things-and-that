# Text Assets

Text-based content for the project.

## Subdirectories

- **scripts/**: Dialogue scripts, cutscene text, and interactive content
- **descriptions/**: Item descriptions, character bios, and informational text
- **lore/**: Background story, world-building content, and narrative elements
- **metadata/**: Configuration files, asset lists, and data structures

## Supported Formats

- **Plain text**: TXT, MD (Markdown)
- **Structured data**: JSON, YAML, XML
- **Rich text**: RTF (when formatting is needed)

## Guidelines

1. Use UTF-8 encoding for all text files
2. Keep line lengths reasonable (80-120 characters)
3. Use Markdown for formatted documentation
4. Include character encoding info when relevant
5. Separate content by language/locale when applicable
6. Use consistent formatting and style
